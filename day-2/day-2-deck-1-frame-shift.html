happy path

What's a state?

A state is what your interface shows in a specific situation.

Same screen, different states. It's the same component and layout, but different things are visible, possible, and available.

You're not designing a screen. You're designing every state that screen can be in.

The thing Figma doesn't teach you

States are about time, not just space

In Figma, you arrange things in space: left, right, padding, alignment, hierarchy.

In a real product, you also design for time: what happens between the user tapping a button and the screen finishing the job.

What does the screen show in that gap? What can the user still do? What if the gap is four seconds because the network is slow? What if it never finishes because the server is down?

Most of the states we'll talk about today are answers to that question: what does the screen show during the in-between?

A shared language for the rest of the day

The core states

Empty

The screen has no data yet. No saved addresses, no cart items, no notifications. Often a design opportunity, not a placeholder.

Loading

Data is on the way. Skeletons, spinners, shimmers. The user should know something is happening.

Partial

Some data is here, more is on the way. Top 5 items show while the next 20 fetch. Real apps live here more than designers expect.

✓

Success

The thing the user wanted has happened. Confirmation, checkmark, next step.

!

Error

Something broke. Wrong input, server failed, network dropped. Has to say what broke and what to do next.

Edge

The weird inputs. 100 items in cart. Username with emojis. Address in a city the app doesn't serve.

Tied to network, permissions, and freshness

The other states worth naming

Offline

The user has no internet. The app should still show something useful. Cached data? A banner? A disabled submit?

Permission denied

The user said no to location, camera, or notifications. Don't break the app — design for refusal.

Stale

Data on screen is old and might be wrong. Cached pricing, expired session, last-synced two hours ago. Tell the user.

Rate-limited / blocked

Too many requests, or the action isn't allowed. Less common, but real on apps that talk to APIs.
