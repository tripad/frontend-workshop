We just spent two hours on states — what your screens need to handle. Now the next question.

The smallest unit of a design system

A token is a named decision

:root {
  --color-accent: #FF6B35;
}

If you've used Figma variables, you've already met tokens. They translate one-to-one to code.

A token is a name attached to a design value.

Instead of writing #FF6B35 in twenty places, you write --color-accent and define it once. The hex is implementation. The name — accent — is the design.

Why it matters:

  • Change the value, twenty places update at once.
  • The name carries intent. accent says why. #FF6B35 only says what.
  • Anyone reading the code sees the system, not a pile of hex codes.

What tokens cover

Color

Palette + roles. Primary, secondary, surface, accent, error, success. Named by purpose, not by hue.

Typography

Font families, scales, weights, line heights. h1, body-md, label-caps. Each name implies use.

Spacing

A small ladder. xs, sm, md, lg, xl. All multiples of a base unit, usually 4 or 8.

Radius

The corner-curve language of your product. sm, md, lg, full. Subtle but enormously consistent.

Shadow

Elevation. card, modal, popover. Each shadow says this is at this layer.

→

Motion

Durations, easings, named transitions. fast, normal, slow. The tempo of the product.

Tokens are the alphabet. Components are the words.

A component isn't "blue rectangle with text." It's a rule: every action in this product looks like this, behaves like this, and you can't accidentally make a different one.

A button is more than its appearance. It includes:

  • Visual style — colors, padding, radius, all from tokens
  • Behavior — hover, focus, active, disabled, loading
  • Variants — primary, secondary, ghost, destructive
  • States — the states from this morning, applied at component level

Use the Button. You inherit all of it.

A vocabulary you'll see in the wild

Atomic Design

Atoms

Smallest pieces. Button, input, label, icon.

Molecules

Small groups. Search bar, form field.

Organisms

Larger sections. Nav bar, product grid, footer.

Templates

Page-level layouts without real content.

Pages

Templates filled with real content.

There's a popular way of categorizing components — Atomic Design, by Brad Frost. You'll see these words in design system docs, articles, and job descriptions.

It's a categorization framework, not a working framework. Useful when you read about design systems. Less useful when you're building one.

Don't worry about which thing is an atom vs a molecule — worry about fewer pieces, named well.

The thing Atomic Design is trying to point at — and the thing that actually matters — is composition.

Components are built from other components. A button uses tokens for color, radius, and type. A search bar uses an input component and a button component. A page uses a nav, a hero section, a card grid, and a footer.

You don't redesign the button when you build the search bar. You don't redraw the search bar when you build the nav. Each layer trusts the layer below it.

This is what makes systems scale — and what makes AI agents useful inside them.

DESIGN.md

Open-sourced April 21, 2026.

DESIGN.md is a single markdown file that lives in your project root.

It contains your design system — tokens, component rules, rationale — in a format any AI agent can read. Drop it in your project. Claude Code, Cursor, Antigravity, Stitch — all of them read it as persistent context.

Every prompt you send is paired with this file before reaching the model.

The result: AI generates UI in your design language by default, not generic defaults.

Google open-sourced the spec on April 21, 2026. You're learning a format the industry hasn't standardized on yet.

## Overview

Architectural minimalism meets journalistic gravitas. The UI evokes a premium matte finish — a high-end broadsheet or contemporary gallery.

A DESIGN.md has two halves.

Top — YAML tokens: machine-readable values the AI parses exactly.

Bottom — Markdown prose: human-readable intent. When no exact rule covers a decision, the AI falls back to the prose.

Together: precise values plus reasoning. That's the file.
