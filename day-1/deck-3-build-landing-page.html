Prompt 1 — Scaffolding

prompt-1 · scaffold.txt 
Create a new folder called `grind-landing` in the current directory. Inside it, create an index.html file with:

- Standard HTML5 boilerplate
- Tailwind CSS loaded via CDN (the script tag version, not a build step)
- Google Font: Instrument Serif (for headings) and Inter (for body)
- A <main> element with a max-width container (max-w-6xl, mx-auto, px-6)
- Background color: #FAF7F2 (warm off-white)
- Body text color: #2B1810 (near-black, warm)

Do not add any page content yet — just the scaffold.

After generating, confirm the file structure and tell me what to run to preview it in the browser.

Prompt 2 — Hero Section

prompt-2 · hero.txt 
Add a hero section to the top of index.html.

Content:
- Small eyebrow text: "A coffee shop for people who care about details."
- Big headline: "Grind."
- Subheadline: "Single-origin coffee. Considered interiors. Free wifi that actually works."
- Primary CTA button: "Find us" (link to #visit)
- Secondary CTA: "See the menu" (text link)

Style:
- Headline uses Instrument Serif, very large (text-8xl+)
- Hero takes full viewport height (min-h-screen)
- Content vertically centered
- Generous spacing
- CTA button: terracotta background (#D4763E), white text, rounded-full, px-8 py-4

Text-only hero, editorial feel.

prompt-3 · about-menu.txt 
Add two sections below the hero.

ABOUT section:
- Heading: "Why Grind" (Instrument Serif, text-5xl)
- Two-column layout on desktop, stacked on mobile
- Left column: a paragraph about the shop's philosophy (single-origin, ethically sourced, designer-friendly)
- Right column: three bullet points:
  • "Ethically sourced, single-origin beans"
  • "Designed for long working sessions"
  • "Quiet hours, 2pm–5pm, Mon–Fri"

MENU section:
- Heading: "The Menu"
- 6 items with prices, restaurant-menu style (dotted leader lines):
  Espresso ₹180, Flat White ₹220, Pour Over ₹280, Cold Brew ₹240, Matcha Latte ₹260, Affogato ₹320
- Italic footnote: "Prices include GST. Beans change weekly."

Both sections: py-24 vertical padding. Warm palette.
prompt-4 · contact-responsive.txt 
Two things.

FIRST — CONTACT section (id="visit"):
- Heading: "Come find us"
- Address: "12, Design District, 2nd Floor. Open 8am–8pm, closed Wednesdays."
- Static map placeholder: a div, bg-stone-200, aspect-video, rounded-2xl, word "Map" centered
- Social links: Instagram, Are.na (text links)

SECOND — responsive pass:
- Test at 375px width
- Hero headline shrinks proportionally (no overflow)
- Menu stacks cleanly
- About columns become one column below md breakpoint
- Sticky nav bar: "Grind" logo left, links right (About, Menu, Visit). Mobile: simple hamburger scrolling to sections.

After this, show me the rendered page and flag anything broken at small widths.
prompt-5 · deploy.txt 
I want to deploy this to GitHub Pages. Walk me through it with exact commands.

Assume I already have a GitHub account (username: [FILL IN]) and git installed.

Give me:
1. Exact terminal commands to init the git repo and make the first commit
2. Exact commands to push to a new repo called `grind-landing`
3. Instructions for enabling GitHub Pages in the repo settings
4. The final URL format my site will be available at

Then do as much of this for me as you can directly from here.
