Why designers who can read code get different answers

When you can read the browser's output — the actual rendered HTML, the computed CSS values, the live DOM — you stop accepting "it can't be done" at face value. You can see why something is hard.

The key insight is that a browser isn't magic. It's parsing a text file, building a tree of objects, and painting pixels. That tree is the DOM. Once you can see it, everything becomes debuggable.

  • Every element is a node in the tree
  • Nodes can be nested — children inside parents
  • CSS selects nodes by tag, class, or ID
  • JavaScript manipulates nodes at runtime

"The best way to understand the web is to break it — in DevTools, where nothing is permanent."

— Nobody in particular, but it's true

Spend 10 minutes right-clicking things you use every day. The gap between "designer" and "developer" gets a lot smaller once you realize both are just reading the same tree.
